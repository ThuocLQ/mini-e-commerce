Role/Claim + Secure Internal Call Intro

Dùng JWT từ IdentityService để bảo vệ endpoint ở CatalogService bằng role/claim. Bài này tập trung đúng scope: validate token, tạo policy AdminOnly, bảo vệ Product write endpoints và test 401/403/201 bằng Postman.

Câu nhớ401 = chưa xác thực hoặc token sai. 403 = đã xác thực nhưng không đủ quyền.

Lý thuyết cốt lõi

Concept

Luồng chính của bài:

Client login
→
IdentityService phát JWT
→
JWT có role Admin
→
CatalogService validate
→
AdminOnly policy
Điểm cần nhớIdentityService phát token. CatalogService chỉ validate token và kiểm tra quyền.

Claim

Thông tin nằm trong token: sub, name, role, iss, aud, exp.

Role

Một claim đặc biệt để nhóm quyền: Admin, Customer, Support.

{
  "sub": "11111111-1111-1111-1111-111111111111",
  "name": "admin",
  "role": "Admin",
  "iss": "MicroShop.IdentityService",
  "aud": "MicroShop.Client"
}

Policy là rule authorization có tên. Trong bài này, policy AdminOnly yêu cầu user có role Admin.

builder.Services.AddAuthorization(options =>
{
    options.AddPolicy("AdminOnly", policy =>
    {
        policy.RequireRole("Admin");
    });
});

Thực hành

Accordion gọn, mở từng bước khi làm.

Build
5.1. Kiểm tra token từ IdentityService có role Admin ⌄

Chạy IdentityService, login bằng Postman, copy accessToken và decode. Token cần có role = Admin.

dotnet run --project Services/IdentityService/IdentityService.csproj
POST http://localhost:5003/auth/login
Content-Type: application/json

{
  "userName": "admin",
  "password": "Admin@123"
}

Nếu chưa có role, kiểm tra trong JwtTokenGenerator:

new(ClaimTypes.Role, user.Role)
5.2. Cài JWT Bearer cho CatalogService ⌄ 
dotnet add Services/CatalogService/CatalogService.csproj package Microsoft.AspNetCore.Authentication.JwtBearer
dotnet build Services/CatalogService/CatalogService.csproj

Package này giúp CatalogService đọc và validate Authorization: Bearer <access_token>.

5.3. Thêm JWT config cho CatalogService ⌄

Sửa Services/CatalogService/appsettings.Development.json:

{
  "ConnectionStrings": {
    "CatalogDb": "Data Source=catalog.db"
  },
  "Jwt": {
    "Issuer": "MicroShop.IdentityService",
    "Audience": "MicroShop.Client",
    "SecretKey": "microshop-development-secret-key-must-be-long-enough"
  }
}

Issuer, Audience, SecretKey phải khớp với IdentityService. Nếu lệch, API trả 401.

5.4. Cấu hình authentication và policy trong CatalogService ⌄

Sửa Services/CatalogService/Program.cs.

using System.Text;
using Microsoft.AspNetCore.Authentication.JwtBearer;
using Microsoft.IdentityModel.Tokens;

var issuer = builder.Configuration["Jwt:Issuer"]
    ?? throw new InvalidOperationException("Jwt:Issuer is missing.");

var audience = builder.Configuration["Jwt:Audience"]
    ?? throw new InvalidOperationException("Jwt:Audience is missing.");

var secretKey = builder.Configuration["Jwt:SecretKey"]
    ?? throw new InvalidOperationException("Jwt:SecretKey is missing.");

builder.Services
    .AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = true,
            ValidIssuer = issuer,
            ValidateAudience = true,
            ValidAudience = audience,
            ValidateIssuerSigningKey = true,
            IssuerSigningKey = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(secretKey)),
            ValidateLifetime = true,
            ClockSkew = TimeSpan.Zero
        };
    });

builder.Services.AddAuthorization(options =>
{
    options.AddPolicy("AdminOnly", policy =>
    {
        policy.RequireRole("Admin");
    });
});
5.5. Thêm middleware đúng thứ tự ⌄ 
var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();

app.MapProductEndpoints();

app.Run();
Cực kỳ quan trọngUseAuthentication() phải đứng trước UseAuthorization().
5.6. Gắn AdminOnly vào ProductEndpoints ⌄

Giữ GET public. Chỉ gắn .RequireAuthorization("AdminOnly") cho POST, PUT, DELETE.

group.MapPost("/", async (
    CreateProductRequest request,
    IValidator<CreateProductCommand> validator,
    ISender sender,
    CancellationToken cancellationToken) =>
{
    var command = new CreateProductCommand(
        request.Name,
        request.Description,
        request.Price);

    var validationResult = await validator.ValidateAsync(command, cancellationToken);

    if (!validationResult.IsValid)
    {
        return Results.ValidationProblem(validationResult.ToDictionary());
    }

    var result = await sender.Send(command, cancellationToken);
    return Results.Created($"/products/{result.Id}", result);
})
.RequireAuthorization("AdminOnly");

Áp dụng tương tự cho PUT và DELETE.

5.7. Build và chạy 2 service ⌄ 
dotnet build Services/CatalogService/CatalogService.csproj
dotnet run --project Services/IdentityService/IdentityService.csproj
dotnet run --project Services/CatalogService/CatalogService.csproj
