PaymentService + Payment Webhook Intro

Tạo PaymentService baseline để mô phỏng thanh toán, tạo payment Pending và nhận webhook success/fail từ provider giả lập. Bài này giúp hiểu webhook khác REST/RabbitMQ/Kafka ở mức nhập môn.

Bắt đầu thực hànhĐi tới Postman Lab

Câu nhớWebhook là HTTP callback từ hệ thống bên ngoài gọi ngược vào hệ thống của mình.

Ý tưởng chính

Hiểu boundary và flow trước khi code.

Concept
ServiceTrách nhiệm
OrderingServiceTạo và quản lý order
DiscountServiceTính discount
PaymentServiceTạo payment, theo dõi payment status, nhận webhook
BasketServiceQuản lý basket

Webhook là HTTP request từ provider gọi về hệ thống của mình khi có kết quả/sự kiện.

REST thường

Mình gọi service khác và chờ response.

Webhook

Provider gọi ngược vào endpoint của mình.

ProductionWebhook thật cần signature, idempotency, webhook log, retry strategy và security.

Thực hành

Mở từng bước để làm. MD có full code chi tiết.

Build
5.1. Tạo PaymentService project ⌄
dotnet new webapi -n PaymentService -o Services/PaymentService
dotnet sln add Services/PaymentService/PaymentService.csproj
dotnet run --project Services/PaymentService/PaymentService.csproj

Port gợi ý: PaymentService = http://localhost:5006.

5.2. Cài MediatR và tạo folder structure ⌄
dotnet add Services/PaymentService/PaymentService.csproj package MediatR
dotnet build Services/PaymentService/PaymentService.csproj
Services/PaymentService/
├── API/Endpoints
├── Application/Abstractions
├── Application/Payments/CreatePayment
├── Application/Payments/GetPaymentById
├── Application/Payments/Webhooks
├── Domain/Payments
└── Infrastructure/Persistence
5.3. Tạo Payment domain model ⌄
public enum PaymentStatus
{
    Pending = 1,
    Succeeded = 2,
    Failed = 3
}
Full codeFile MD có full class Payment gồm constructor, MarkSucceeded và MarkFailed.
5.4. Tạo DTO, mapper, repository ⌄
public sealed record PaymentDto(
    Guid Id,
    Guid OrderId,
    Guid CustomerId,
    decimal Amount,
    string Currency,
    string Status,
    string? ProviderTransactionId,
    string? FailureReason,
    DateTime CreatedAtUtc,
    DateTime? CompletedAtUtc);
public interface IPaymentRepository
{
    Task<Payment> CreateAsync(Payment payment, CancellationToken cancellationToken = default);
    Task<Payment?> GetByIdAsync(Guid id, CancellationToken cancellationToken = default);
    Task<bool> UpdateAsync(Payment payment, CancellationToken cancellationToken = default);
}
5.5. Tạo use cases ⌄
public sealed record CreatePaymentCommand(
    Guid OrderId,
    Guid CustomerId,
    decimal Amount,
    string Currency) : IRequest<PaymentDto>;

public sealed record GetPaymentByIdQuery(Guid Id) : IRequest<PaymentDto?>;

public sealed record PaymentWebhookCommand(
    Guid PaymentId,
    string ProviderTransactionId,
    string Status,
    string? FailureReason) : IRequest<PaymentDto?>;
5.6. Tạo endpoints và Program.cs ⌄
builder.Services.AddSingleton<IPaymentRepository, InMemoryPaymentRepository>();

app.MapPaymentEndpoints();
app.MapWebhookEndpoints();
